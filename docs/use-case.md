# Admin

## 상품관리

### 1. 상품 목록
- 유즈케이스 이름: 상품 목록 조회하기
- 액터: 관리자
- 목표: 관리자가 상품 목록을 조회한다.
- 사전 조건: 
  - 관리자 계정으로 로그인 되어 있어야 한다.
  - 관리자가 상품 관리 페이지에 접속되어 있어야 한다.
- 기본 흐름
- 기본 흐름
  1. 상품 목록이 조회된다. (상품명, 카테고리, 가격, 재고, 상태, 판매량)
  1-1. 관리자가 검색어를 입력하거나 필터 조건(상태, 카테고리 등)을 선택한다.
  1-2. 시스템은 조건에 맞는 상품 목록만 조회하여 표시한다.
- 예외 흐름
    - 권한이 없는 사용자가 요청하면 접근 거부 메시지를 보여준다.
    - 등록된 상품이 없으면 “조회 가능한 상품이 없습니다” 메시지를 보여준다.
    - DB 조회 중 오류가 발생하면 에러 메시지를 보여준다.


### 2. 상품 추가
- 유즈케이스 이름: 상품 추가하기
- 액터: 관리자
- 목표: 관리자가 상품을 등록한다.
- 사전 조건: 
  - 관리자 계정으로 로그인 되어 있어야 한다.
  - 관리자가 상품 관리 페이지에 접속되어 있어야 한다.
- 기본 흐름
  1. ‘상품 추가’ 버튼을 누른다.
  2. 상품 추가에 필요한 항목을 입력한다. (*상품명, *판매가격, *정가, *카테고리, *상태, *재고 수량, 재고 경고 기준, 이미지, 상품 설명, 주요 특징)
  3. ‘추가’ 버튼을 누른다.
- 예외 흐름
    - 권한이 없는 사용자가 요청하면 접근 거부 메시지를 보여준다.
    - 필수 입력 항목이 누락되면 시스템은 에러 메세지를 보여주고, 등록을 거부한다.
    - 동일한 상품명이 이미 존재하면 시스템은 중복 경고 메세지를 보여준다.
- 사후 조건
    - 시스템은 상품을 등록하고, 새로운 상품이 상품 목록에 표시된다.


### 3. 상품 수정
- 유즈케이스 이름: 상품 수정하기
- 액터: 관리자
- 목표: 관리자가 상품을 수정한다.
- 사전 조건: 
  - 관리자 계정으로 로그인 되어 있어야 한다.
  - 관리자가 상품 관리 페이지에 접속되어 있어야 한다.
- 기본 흐름
    1. 수정하고 싶은 상품의 ‘수정하기’ 버튼을 클릭한다.
    2. 수정이 필요한 항목의 값을 변경한다.
    3. ‘수정’버튼을 누른다.
- 예외 흐름
    - 권한이 없는 사용자가 요청하면 접근 거부 메시지를 보여준다.
    - 필수 입력 항목이 누락되면 시스템은 에러 메세지를 보여주고, 수정을 거부한다.
    - 동일한 상품명이 이미 존재하면 시스템은 중복 경고 메세지를 보여준다.
    - 존재하지 않는 상품을 수정하려 하면 에러 메시지를 보여준다.
  - 사후 조건
    - 시스템은 상품 정보를 수정하고, 수정한 정보가 상품 목록에 노출된다.


### 4. 상품 삭제
- 유즈케이스 이름: 상품 삭제하기
- 액터: 관리자
- 목표: 관리자가 기존 상품을 삭제한다.
- 사전 조건
    - 관리자 계정으로 로그인되어 있어야 한다.
    - 관리자가 상품 관리 페이지에 접속되어 있어야 한다.
- 기본 흐름:
    1. 삭제할 상품의 '삭제하기' 버튼을 클릭한다.
    2. 시스템은 삭제 확인(confirm) 창을 노출한다.
    3. 관리자가 '확인'을 클릭한다.
- 예외 흐름:
    - 권한이 없는 사용자가 요청하면 접근 거부 메시지를 보여준다.
    - 관리자가 confirm 창에서 '취소'를 누르면 삭제가 진행되지 않는다.
    - 존재하지 않는 상품을 삭제하려 하면 에러 메시지를 보여준다.
- 사후 조건:
    - 시스템은 상품 데이터를 삭제하고, 상품 목록에서 해당 상품이 비노출된다.

  2. 시스템은 해당 상품의 수량을 변경하고, 총 결제 금액을 업데이트한다.
- 예외 흐름:
  - 권한이 없는 사용자가 접근하면 “로그인 후 이용 가능합니다.” 메시지를 보여준다.
  - 재고보다 큰 수량을 입력하면 “재고가 부족합니다.” 메시지를 보여준다.
  - 수량이 0 이하일 경우 “올바른 수량을 입력하세요.” 메시지를 보여준다.
- 사후 조건: 
  - 변경된 수량과 총 결제 금액이 장바구니에 반영된다.  


### 3. 장바구니 상품 삭제
- 유즈케이스 이름: 장바구니 상품 삭제하기
- 액터: 고객
- 목표: 고객이 장바구니에서 불필요한 상품을 삭제한다. 
- 사전 조건:
  - 로그인이 되어 있어야 한다.
  - 장바구니 페이지에 접속되어 있어야 한다.
- 기본 흐름:
  1. 로그인이 되어 있어야 한다.
  2. 고객이 특정 상품의 "삭제" 버튼을 클릭한다.
  3. 시스템은 해당 상품을 장바구니에서 제거하고 목록을 갱신한다. 
- 예외 흐름:
  - 권한이 없는 사용자가 접근하면 “로그인 후 이용 가능합니다.” 메시지를 보여준다.
  - 삭제하려는 상품이 이미 존재하지 않는 경우 “삭제할 상품이 없습니다.” 메시지를 보여준다. 
- 사후 조건: 
  - 선택한 상품이 장바구니에서 제거된다.

### 4. 상품 주문
- 유즈케이스 이름: 장바구니 상품 주문하기
- 액터: 고객
- 목표: 고객이 장바구니에 담긴 상품을 주문한다.
- 사전 조건:
  - 로그인이 되어 있어야 한다.
  - 장바구니 페이지에 접속되어 있어야 한다.
- 기본 흐름:
  1. 고객이 장바구니에서 주문할 상품을 선택한다.
  2. 주문 요약 영역에서 “주문하기” 버튼을 클릭한다.
- 예외 흐름:
  - 권한이 없는 사용자가 접근하면 “로그인 후 이용 가능합니다.” 메시지를 보여준다.
  - 선택된 상품이 없으면 “주문할 상품을 선택하세요.” 메시지를 보여준다.
  - 품절된 상품을 포함하면 “품절 상품은 주문할 수 없습니다.” 메시지를 보여준다.
- 사후 조건:
  - 결제 페이지로 이동한다.


## 주문관리

### 1. 주문 목록 조회
- 유즈케이스 이름: 주문 목록 조회하기
- 액터: 고객
- 목표: 고객이 본인의 주문 내역을 조회한다.
- 사전 조건: 
  - 로그인이 되어 있어야 한다.
  - 주문 목록 페이지에 접속되어 있어야 한다.
- 기본 흐름:
  1. 전체 주문 건수를 집계되어 노출된다. (전체, 진행 중, 완료, 취소)
  2. 주문한 상품 목록이 노출된다.
- 예외 흐름:
  - 권한이 없는 사용자가 접근하면 “로그인 후 이용 가능합니다.” 메시지를 보여준다.
  - 주문 내역이 없으면 “주문 내역이 존재하지 않습니다.” 메시지를 보여준다.


### 2. 주문 취소
- 유즈케이스 이름: 주문 취소하기
- 액터: 고객
- 목표: 고객이 주문을 취소한다.
- 사전 조건:
  - 로그인이 되어 있어야 한다.
  - 주문 상태가 ‘결제 대기’ 또는 ‘처리 전’이어야 한다.
  - 주문 목록 페이지에 접속되어 있어야 한다.
- 기본 흐름:
  1. 고객이 주문 목록에서 특정 주문의 "주문 취소" 버튼을 클릭한다.
  2. 시스템은 주문 취소 확인 창을 노출한다.
  3. 고객이 ‘확인’을 클릭한다.
- 예외 흐름:
  - 권한이 없는 사용자가 접근하면 “로그인 후 이용 가능합니다.” 메시지를 보여준다.
  - 이미 배송 중 또는 완료된 주문은 취소할 수 없으며, “현재 상태에서는 취소가 불가능합니다.” 메시지를 보여준다.
  - 존재하지 않는 주문을 취소하려고 하면 “해당 주문을 찾을 수 없습니다.” 메시지를 보여준다.
- 사후 조건: 
  - 주문 목록에 ‘주문 취소’ 상태로 반영된다.
